⚠ Strona robocza — nie wysyłaj klientom

1. Dane klienta

Identyfikator folderu (slug) generowany automatycznie z nazwiska/imienia inwestora i adresu budowy. Plik manifestu zapisze się jako klienci/<slug>/manifest.json.
slug: (uzupełnij inwestora i adres)

2. Rysunki do akceptacji (slajdy)

Każdy rysunek = jeden slajd. Kolejność na liście = kolejność prezentacji klientowi. Akceptowane są PDF jednostronicowe. Kliknij „📄 Wybierz plik" by wskazać PDF z dysku.

3. Zapis na dysk + push na GitHub

Wybierz folder klienci/ w lokalnym repozytorium. Admin utworzy w nim podfolder klienta z plikami PDF i manifestem. Potem ręcznie wykonaj git add . && git commit -m "..." && git push (lub kliknij „Pokaż link" by skopiować URL gotowej ankiety).

4. Istniejący klienci (z folderu)

Lista folderów już zapisanych w katalogu klienci/. Wybierz aby otworzyć ankietę lub edytować.

Kliknij „Odśwież listę" aby wskazać folder klienci/.
